Pydantic AI 기반 RAG 검색 챗봇 설계서

생성 AI + 문서 업로드 + 벡터 검색 기반 지능형 문서 QA 시스템

1. 시스템 개요

본 시스템은 Pydantic AI 에이전트 프레임워크를 활용하여 사용자가 업로드한 문서를 벡터 임베딩으로 인덱싱하고, 자연어 질문에 대해 문서 기반 답변을 생성하는 RAG(Retrieval-Augmented Generation) 챗봇입니다.

UI/UX는 Claude Cowork 스타일을 따라 따뜻한 색감의 믈니멀 디자인, 사이드바 문서 관리, 컨텍스트 패널, 타이핑 인디케이터 등을 제공합니다.

2. 기술 스택

Pydantic AI (v1.x)

에이전트 프레임워크. 타입 안전성, 의존성 주입, 도구 등록, 구조화된 출력을 제공. OpenAI/Anthropic/Gemini 등 모델 무관 지원.

FastAPI

비동기 REST API 서버. SSE 스트리밍 응답, 파일 업로드 엔드포인트, 세션 관리를 처리.

PostgreSQL + pgvector

벡터 저장소. 코사인 유사도 기반 시맨틱 검색 수행. 하이브리드 검샩(벡터 + 키워드) 지원.

OpenAI Embeddings

text-embedding-3-small 모델로 문서 청크와 질의를 1536차원 벡터로 변환.

LangChain (문서처리)

PDF/DOCX/TXT 등 다양한 포맷의 문서 로딩과 시맨틱 청킹 담당.

React + Tailwind

Claude Cowork 스타일 프론트엔드. 문서 드래그&드롭, 실시간 스트리밍 응답 UI.

4. 핵심 모듈 설계

4.1 Pydantic AI 에이전트 정의

from pydantic_ai import Agent from pydantic import BaseModel from dataclasses import dataclass @dataclass class RAGDeps: """에이전트 의존성: DB 연결 + 임베딩 클라이언트""" pool: AsyncConnectionPool openai_client: AsyncOpenAI class ChatResponse(BaseModel): """구조화된 응답 스키마""" answer: str sources: list[str] confidence: float rag_agent = Agent( 'openai:gpt-4o', # 모델 무관 — Anthropic, Gemini 등 교체 가능 deps_type=RAGDeps, output_type=ChatResponse, instructions="""업로드된 문서를 검색하여 정확한 답변을 제공합니다. 반드시 retrieve 도구를 사용하여 관련 문서를 찾은 후 답변하세요. 답변에 출처를 반드시 폨함하세요.""" )

4.2 검색 도구(Tool) 등록

@rag_agent.tool async def retrieve(ctx: RunContext[RAGDeps], query: str) -> str: """벡터 유사도 검색으로 관련 문서 청크를 반환""" # 1. 질의를 임베딩 벡터로 변환 embedding = await ctx.deps.openai_client.embeddings.create( model='text-embedding-3-small', input=query ) query_vector = embedding.data[0].embedding # 2. pgvector 코사인 유사도 검색 rows = await ctx.deps.pool.fetch(""" SELECT content, doc_name, similarity FROM chunks ORDER BY embedding <-> $1::vector LIMIT 5 """, query_vector) # 3. 검색 결과를 컨텍스트로 조합 return '\n---\n'.join( f"[{r['doc_name']}]: {r['content']}" for r in rows )

4.3 문서 처리 파이프라인

1
업로드 수신 FastAPI /upload 엔드포인트에서 PDF, DOCX, TXT, MD 파일 수신
2
텍스트 추출 Docling(PDF 테이블/이미지), python-docx(DOCX), 직접 파싱(TXT/MD)
3
시맨틱 청킹 RecursiveCharacterTextSplitter로 500~1000자 단위 분할, 100자 오버랩
4
임베딩 생성 각 청크를 text-embedding-3-small로 1536차원 벡터 변환, 배치 처리
5
인덱싱 저장 PostgreSQL pgvector에 벡터 + 메타데이터 저장, HNSW 인덱스 적용

4.4 API 엔드포인트 설계

# FastAPI 라우트 정의 @app.post("/api/upload") async def upload_document(file: UploadFile): """문서 업로드 → 추출 → 청킹 → 임베딩 → 저장""" @app.post("/api/chat") async def chat(request: ChatRequest): """동기 채팅 — Pydantic AI 에이전트 실행""" @app.post("/api/chat/stream") async def chat_stream(request: ChatRequest): """SSE 스트리밍 — 실시간 토큰 단위 응답""" @app.get("/api/documents") async def list_documents(): """업로드된 문서 목록 + 인덱싱 상태""" @app.delete("/api/documents/{doc_id}") async def delete_document(doc_id: str): """문서 및 관련 청크 삭제""" @app.get("/api/search") async def search(query: str, mode: str = "hybrid"): """직접 검색 — vector / hybrid 모드"""

5. DB 스키마

-- PostgreSQL + pgvector 확장 CREATE EXTENSION IF NOT EXISTS vector; CREATE TABLE documents ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), name TEXT NOT NULL, file_type TEXT NOT NULL, file_size INTEGER, chunk_count INTEGER DEFAULT 0, status TEXT DEFAULT 'processing', -- processing | indexed | error created_at TIMESTAMPTZ DEFAULT now() ); CREATE TABLE chunks ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), doc_id UUID REFERENCES documents(id) ON DELETE CASCADE, content TEXT NOT NULL, embedding VECTOR(1536) NOT NULL, chunk_index INTEGER, metadata JSONB DEFAULT '{}', created_at TIMESTAMPTZ DEFAULT now() ); -- HNSW 인덱스: 빠른 근사 최근접 탐색 CREATE INDEX idx_chunks_embedding ON chunks USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64); CREATE TABLE conversations ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), session_id TEXT NOT NULL, role TEXT NOT NULL, -- user | assistant content TEXT NOT NULL, sources JSONB DEFAULT '[]', created_at TIMESTAMPTZ DEFAULT now() );

6. Pydantic AI 핵심 장점 (vs LangChain)

타입 안전성

Pydantic 기반 입출력 검증. IDE 자동완성, 런타임 전 오류 검출. LangChain 대비 오류율 현저히 감소.

토큰 효율성

LangChain 대비 2.7배 적은 토큰 소비(동일 RAG 작업 기준). 불필요한 추상화 레이어 제거.

의존성 주입

테스트와 프로덕션 환경에서 동일 에이전트 코드 재사용. DB/API 클라이언트를 깔끔하게 주입.

모델 교체 용이

OpenAI → Anthropic → Gemini 등 한 줄 변경으로 모델 교체. 벤더 종속 없음.
